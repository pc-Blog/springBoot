# 个人博客网站 — 功能需求分析（已确认）

## 项目定位
面向程序员的个人技术博客，核心目标：**展示学习成长轨迹** + **项目作品集**。

## 已确认的关键决策
| 决策项 | 选择 |
|--------|------|
| 评论系统 | 需要，无需审核，访客评论直接展示 |
| SEO | 不做 SEO，纯 SPA 客户端渲染 |
| 部署环境 | 单一云服务器（前端 + 后端 + 数据库同一台） |
| 内容初始化 | 从零开始，无需迁移 |
| 外部数据源 | 本地 Tomcat 服务器提供文学作品 + 音乐资源 |
| 数据集成方式 | SpringBoot 后端代理 Tomcat API，统一接口输出给前端 |
| 前端框架 | Next.js + TypeScript |
| 后端框架 | SpringBoot + Java 21 |

### 2.6 音乐播放 ✨ (数据来源：Tomcat)
| 功能点 | 说明 |
|--------|------|
| 迷你播放器 | 页面底部/侧边固定迷你条，仅展示当前曲目名、歌手 |
| 随机播放 | 随机切歌，没有上一首/列表/循环等复杂控制 |
| 播放/暂停 | 一个按钮切换播放和暂停状态 |
| 封面小图 | 迷你播放器旁展示当前曲目封面缩略图（可选） |

> **设计原则**：极简，不干扰阅读。背景音乐式的存在，用户不用管"选哪首"。  
> **数据流**：前端 → SpringBoot(代理层) → Tomcat 随机曲目接口 → 返回单曲信息 + MinIO URL  
> **播放方式**：前端 `<audio>` 直接加载 MinIO URL  
> **管理说明**：音乐的上传/管理在 Tomcat + MinIO 侧完成

## 四、非功能性需求

- **性能**：图片懒加载、音频流式加载不阻塞页面
- **安全**：防 XSS（评论/Markdown 内容）、防 CSRF、管理接口鉴权
- **可维护**：前后端分离，Tomcat 代理层独立封装，便于切换数据源
- **容错**：Tomcat 不可用时，文学/音乐模块优雅降级（显示不可用提示，不影响博客核心功能）

## 六、Tomcat 集成策略（已确认）

| 决策项 | 选择 |
|--------|------|
| 部署关系 | Tomcat 与 SpringBoot **同一台机器** |
| 接口现状 | Tomcat **尚未**有适配博客的接口，后期补充 |
| 音乐存储 | 文件存 **MinIO**，Tomcat 返回 MinIO URL |
| 音频播放 | 前端 `<audio>` 直接加载 MinIO URL 流式播放 |
| MVP 策略 | Tomcat 接口未就绪期间，SpringBoot 提供 Mock 数据，保证前端可独立开发调试 |

### 数据流架构

```
┌──────────┐   请求     ┌──────────┐   代理转发   ┌──────────┐
│ Next.js前端  │ ────────→ │ SpringBoot│ ──────────→ │  Tomcat  │
│          │ ←──────── │          │ ←────────── │ (文学/音乐)│
└────┬─────┘   JSON    └──────────┘   JSON      └──────────┘
     │
     │ 音频直连
     ↓
  ┌───────┐
  │ MinIO │ (音乐文件存储)
  └───────┘
```

- **文学作品**：Next.js → SpringBoot → Tomcat → 返回文学内容 JSON
- **音乐元数据**：Next.js → SpringBoot → Tomcat → 返回曲目列表 JSON（含 MinIO URL）
- **音频流**：Next.js `<audio>` 直接请求 MinIO URL 播放（绕过 SpringBoot，避免代理流量压力）

### Tomcat 接口契约（待定义，供后续 Tomcat 端开发参考）

博客项目需与 Tomcat 约定以下接口：

**文学接口：**
```
GET  /api/literature/list?type=poem|prose&page=1&size=10  → 作品列表
GET  /api/literature/{id}                                   → 作品详情
GET  /api/literature/types                                   → 所有体裁
```

**音乐接口：**
```
GET  /api/music/random                     → 随机返回一首曲目（曲名、歌手、封面、MinIO URL）
```
